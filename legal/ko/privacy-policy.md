**서비스 제공업체와 공유하는 데이터:**
- **익명/가명 데이터:**
  - 집계 사용 통계

**일반 서비스 제공업체와 공유하지 않는 정보:**
- 이름, 이메일 또는 연락처 정보 (서비스 제공에 필요한 경우 제외, 예: 이메일 제공업체)
- 개인 식별 정보 (PII)
- 귀하의 신원과 연결된 개별 세션 원시 데이터

**계약적 보호:**
- 제3자 제공업체는 계약에 따라 다음을 준수해야 합니다:
  - 당사가 지정한 목적으로만 데이터 사용
  - 적절한 보안 조치 시행
  - 적용 가능한 개인정보 보호법 준수 (GDPR, CCPA)
  - 다른 당사자와 데이터 판매 또는 공유 금지
  - 더 이상 필요하지 않을 때 데이터 삭제

### 5.4 익명화 기준

**당사는 귀하의 개인정보를 보호하기 위해 엄격한 익명화 기법을 적용합니다:**

**비식별화:**
- 모든 직접 식별자가 영구적으로 제거됩니다

**집계:**
- 가능한 경우 다수의 사용자에 걸쳐 데이터가 집계됩니다
- 최소 그룹 크기 (k-익명성)가 적용됩니다
- 통계적 공개 제어 기법이 적용됩니다

**데이터 최소화:**

- 가능한 경우 세분화가 줄어듭니다 (예: 초단위 대신 일단위 데이터)

**재식별 위험 평가:**
- 공개적으로 이용 가능한 데이터셋과 결합 시 재식별이 불가능한지 데이터를 검토합니다
- 공유 전 통계적 위험 분석이 수행됩니다
- 고위험 데이터셋의 경우 외부 개인정보 보호 전문가의 자문을 구할 수 있습니다

### 5.5 연구 데이터 사용 계약

**데이터를 수령하는 연구자는 다음을 준수해야 합니다:**

**데이터 사용 계약(DUA)에 서명하여 다음을 요구합니다:**
- 지정된 연구 목적으로만 데이터 사용
- 개인 재식별 시도 금지
- 승인 없이 다른 당사자와 데이터 공유 금지
- 적절한 보안 조치 시행
- 연구 완료 시 데이터 삭제 또는 반환
- 연구 윤리 기준 준수

**기관생명윤리위원회 (IRB) 승인 취득:**
- 인간 대상 연구는 IRB의 검토 및 승인을 받아야 합니다
- 연구 수행에 대한 윤리 기준을 충족해야 합니다
- 취약 집단에 대한 추가 보호 조치

**적용 가능한 법률 준수:**
- GDPR (일반 데이터 보호 규정) - EU 거주자용
- CCPA (캘리포니아 소비자 개인정보 보호법) - 캘리포니아 거주자용
- HIPAA (건강 데이터가 보호 건강 정보로 간주되는 경우)
- 연구 관련 규정 (Common Rule 등)

**연구 결과의 책임 있는 공개:**
- 연구 결과가 재식별을 가능하게 해서는 안 됩니다
- 집계 수준의 결과만 공개
- 개별 데이터 포인트는 게시되지 않습니다

### 5.6 연구 데이터 공유 거부

**귀하는 언제든지 연구 데이터 공유를 거부할 권리가 있습니다.**

**거부 방법:**
1. 계정 설정 → 개인정보 설정으로 이동
2. "연구를 위한 익명 데이터 공유 허용"을 끄기로 전환
3. 기본 설정이 즉시 저장됩니다

**거부의 효과:**
- 귀하의 데이터는 향후 연구 데이터셋에 포함되지 않습니다
- 이미 익명화된 형태로 공유된 데이터는 회수할 수 없습니다 (익명화로 인해 귀하의 기록을 식별할 수 없음)
- 거부해도 Ultiracer 서비스 사용 능력에 영향을 미치지 않습니다
- 언제든지 다시 참여할 수 있습니다

**기본 설정:**
- 연구 데이터 공유는 기본적으로 옵트인입니다 (적극적으로 동의해야 함)
- 또는 이용약관 동의에 포함된 경우 언제든지 거부할 수 있습니다

### 5.7 연구 데이터 공유의 이점

**연구 데이터 공유를 수행하는 이유:**
- 운동, 건강 및 인간 웰니스에 대한 과학적 이해 증진
- 공중 보건 이니셔티브 기여
- 더 나은 웰니스 방법 및 건강 개입 개발 지원
- 삶을 개선할 수 있는 의학 연구 지원
- 근거 기반 피트니스 및 건강 권장 사항 지원

**연구에 사용된 데이터에 대한 보상은 제공되지 않습니다**, 그러나 사회에 이익이 되는 과학 발전에 기여하게 됩니다.

## 8. 귀하의 권리와 선택

### 8.1 GDPR 권리 (EU 거주자)

유럽 연합에 거주하는 경우 다음 권리가 있습니다:

- **접근권:** 개인 데이터 사본 요청
- **정정권:** 부정확하거나 불완전한 데이터 수정
- **삭제권 ("잊힐 권리"):** 개인 데이터 삭제
- **처리 제한권:** 데이터 사용 방법 제한
- **데이터 이동권:** 구조화되고 기계 판독 가능한 형식으로 데이터 수령
- **이의 제기권:** 정당한 이익 또는 직접 마케팅에 기반한 처리에 이의 제기
- **동의 철회권:** 언제든지 동의 철회 (이전 처리에 영향 없음)
- **불만 제기권:** 해당 국가의 데이터 보호 기관에 불만 제기

**권리 행사를 위한 연락처:** privacy@rdt.ai

### 8.2 CCPA 권리 (캘리포니아 거주자)

캘리포니아 거주자인 경우 다음 권리가 있습니다:

- **알 권리:** 수집, 사용, 공개 및 판매하는 개인정보
- **삭제권:** 개인 데이터 삭제 요청 (예외 있음)
- **판매 거부권:** 당사는 귀하의 개인정보를 판매하지 않습니다
- **차별 금지권:** 권리 행사에 대해 차별하지 않습니다

**권리 행사를 위한 연락처:** privacy@rdt.ai로 연락

### 8.3 계정 설정 및 기본 설정

계정을 통해 개인정보 설정을 제어할 수 있습니다:

- **프로필 정보 편집:** 이름, 이메일, 전화번호 등 업데이트
- **개인정보 설정:** 데이터 공유 기본 설정 제어
- **알림 기본 설정:** 이메일 및 푸시 알림 관리
- **연결된 앱:** 제3자 통합 관리
- **데이터 내보내기:** JSON 형식으로 데이터 다운로드 (GDPR 준수)
- **계정 삭제:** 계정 및 데이터 영구 삭제

### 8.4 마케팅 커뮤니케이션

마케팅 이메일 수신을 거부할 수 있는 방법:
- 마케팅 이메일의 "구독 취소" 클릭
- 계정 설정 → 알림에서 기본 설정 조정

**참고:** 계정을 삭제하지 않고는 서비스 관련 커뮤니케이션 (예: 보안 알림, 계정 알림)의 수신을 거부할 수 없습니다.

### 8.5 추적 금지 (DNT)

일부 브라우저는 "추적 금지" (DNT) 신호를 제공합니다. 현재 DNT 신호를 처리하기 위한 업계 표준이 없으므로 DNT 신호에 응답하지 않습니다.

## 9. 보안 조치

당사는 귀하의 정보를 보호하기 위해 업계 표준 보안 조치를 시행합니다:

**기술적 보호 조치:**
- **전송 중 암호화:** 귀하의 기기와 서버 간에 전송되는 모든 데이터는 TLS 1.3을 사용하여 암호화됩니다
- **저장 시 암호화:** 민감한 데이터는 AES-256을 사용하여 데이터베이스에 암호화됩니다
- **보안 인증:** 비밀번호는 솔트가 포함된 bcrypt를 사용하여 해시됩니다
- **다중 인증 (MFA):** 계정 보호를 위해 사용 가능
- **정기 보안 감사:** 침투 테스트 및 취약성 평가

**조직적 보호 조치:**
- **접근 제어:** 직원은 업무에 필요한 데이터에만 접근 가능
- **신원 조회:** 데이터에 접근하는 직원은 신원 조회를 거침
- **보안 교육:** 모든 직원에 대한 정기적인 개인정보 보호 및 보안 교육
- **사고 대응 계획:** 침해 감지, 대응 및 보고를 위한 절차
- **제3자 보안 평가:** 벤더의 보안 관행 검증

**물리적 보호 조치:**
- **안전한 데이터 센터:** 물리적 접근 제어가 있는 업계 인증 시설
- **이중화 및 백업:** 손실 방지를 위한 정기적인 데이터 백업

**제한 사항:**
- 어떤 보안 시스템도 완벽하지 않습니다. 절대적인 보안을 보장할 수 없습니다.
- 계정 자격 증명의 보안 유지는 귀하의 책임입니다.
- 의심되는 보안 문제는 security@rdt.ai으로 보고해 주세요

**데이터 침해 알림:**
- 데이터 침해 발생 시 영향을 받는 사용자에게 72시간 이내에 알립니다 (GDPR 요구 사항)
- 법률이 요구하는 대로 관련 당국에 알립니다
- 침해에 대한 정보 및 자신을 보호하기 위한 조치를 제공합니다
